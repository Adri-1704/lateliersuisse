## Principes Architecturaux

1. **Simplicite d'abord** (KISS, YAGNI) - pas d'over-engineering
2. **Type-safety partout** - TypeScript strict, schemas Zod
3. **Server Components par defaut** - Client Components uniquement quand necessaire
4. **Edge-first** pour les performances en Suisse/Europe
5. **Convention over configuration** - suivre les patterns Next.js/Supabase
6. **Separation des responsabilites** - chaque module a une seule raison de changer
